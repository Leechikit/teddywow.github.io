获取对象大小

elem.offsetWidth/offsetHeight

获取对象的可见宽度和高度，包括滚动条、边框、内边距，会随元素的显示大小改变。

  • 当对象为内联元素时，offsetWidth和offsetHeight可以得到正确的值。
  • document.body.clientWidth/clientHeight
    获取BODY对象宽度和高度，随内容改变。
  • document.documentElement.clientWidth/clientHeight
    获取浏览器窗口可见区域宽度和高度，不随内容改变。

如下图红框为所示区域

elem.clientWidth/clientHeight

获取对象可见的宽度和高度，不包括滚动条、边框，包括内边距，会随元素的显示大小改变。

  • 当对象为内联元素时，clientWidth和clientHeight无法得到值，显示为0。

如下图红框为所示区域

elem.scrollWidth/scrollHeight

获取对象的实际内容的宽度和高度，不包括滚动条、边框，包括内边距，会随对象中内容的多少改变（内容多了可能会改变对象的实际宽度和高度）。

  • 当对象为内联元素时，scrollWidth和scrollHeight无法得到值，显示为0。

如下图红框为所示区域，红框为实际内容展开的大小。

window.screen.width/screen.height

获取屏幕分辨率的宽度和高度，包括滚动条，固定不变，与浏览器大小无关。

window.screen.availWidth/screen.availHeight

获取屏幕可用工作区宽度和高度，包括滚动条，固定不变，与浏览器大小无关。

window.innerWidth/innerHeight

获取浏览器窗口可见区域宽度和高度，不随内容改变。

比较获取浏览器窗口可见区域大小的方法

  • 在PC端浏览器上document.documentElement.clientWidth/clientHeight和window.innerWidth/innerHeight获取的浏览器窗口可见区域宽度和高度一样
  • 在WAP端中，大多浏览器上两个方法获取的宽高都一样，但是在ios上的UC和QQ浏览器上，因为地址栏可伸缩隐藏，导致两个方法获取的高度不一样。document.documentElement.clientHeight获取到的是地址栏隐藏时的窗口高度（与html和body高度设为100%时一致），而window.innerHeight获取到的是地址栏未隐藏时的窗口高度。
  • 建议使用document.documentElement.clientWidth/clientHeight获取浏览器窗口可见区域宽度和高度。

获取对象定位

offsetParent

offsetParent属性返回一个对象的引用，这个对象是距离调用offsetParent的元素最近的（在包含层次中最靠近的），并且是已进行用过CSS定位的容器元素。如果这个容器元素未进行CSS定位, 则offsetParent属性的取值为根元素body的引用。

  • 当某个元素及其DOM结构层次中元素都未进行CSS定位时(absolute或者relative)，则这个元素的offsetParent属性的取值为根元素body。更确切地说，这个元素的各种偏移量计算（offsetTop、offsetLeft等）的参照物为根元素。
  • 当某个元素的父元素进行了CSS定位时（absolute或者relative），则这个元素的offsetParent属性的取值为其父元素。更确切地说，这个元素的各种偏移量计算（offsetTop、offsetLeft等）的参照物为其父元素。
  • 当某个元素CSS定位为fixed时，元素的offsetParent属性返回null.
  • 当容器元素的style.display 被设置为 none时（译注：IE和Opera除外），offsetParent属性返回null。

elem.offsetLeft/offsetTop

获取对象（从边框开始）相对于版面或由offsetParent 属性指定的父坐标（从边框开始）的计算左侧和上侧位置。

如下图两个红箭头分别是box的offsetLeft和offsetTop所获得的距离，注意container已设为position:relative，不包括container和box的边框。

elem.scrollLeft/scrollTop

设置或获取位于对象左边界/上边界和窗口中目前可见内容的最左端/最上端之间的距离。

  • document.documentElement.scrollTop获取页面垂直方向滚动的值。

如下图，滚动条向下滚动一半，两个红箭头分别是container的scrollLeft和scrollLeft所获得的距离，因为container没有向左滚动，所以scrollLeft为0，而container的滚动条向下滚动中间，所以scrollTop为container向上滚动的距离。

window.screenLeft/screenTop

获取浏览器距离屏幕的左端和上端的距离。

event.clientX/Y

获取相对浏览器可视区域的座标，不计算滚动到可视区域外的页面部分。

如下图两个红箭头分别是触发事件对象的clientX和clientY所获得的距离

event.offsetX/Y

获取相对容器内部的坐标。

如下图两个红箭头分别是触发事件对象的offsetX和offsetY所获得的距离

event.screenX/Y

获取相对屏幕的坐标。

event.pageX/Y

获取相对整个页面内部的坐标，包括滚动到可视区域外的页面部分。

如下图两个红箭头分别是触发事件对象的pageX和pageY所获得的距离
