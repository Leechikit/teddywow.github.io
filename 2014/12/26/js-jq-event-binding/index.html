addEventListener()

语法：

除IE外的现代浏览器的绑定事件方法，可控制事件传播的方式。

  • useCapture：定在捕捉过程还是冒泡过程执行事件，true为在捕捉过程，false为在冒泡过程，默认为false。

attachEvent()

微软的私有方法，为元素绑定事件。
语法：

  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • function：定义当事件发生时运行的方法。

detachEvent()

语法：

detachEvent()方法只能注销通过attachEvent()绑定的事件。
注销事件时detachEvent的两个个参数必须要跟attachEvent的参数相同，否则会出错。这里则要求要注销的事件在用attachEvent绑定时function不能是匿名函数。

jQuery事件绑定方法

bind()

为每个匹配元素的特定事件绑定事件处理函数。
语法：

  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • data：可选。传递给事件对象的额外数据对象。
  • function：定义当事件发生时运行的方法。

优点：

  1. 提供了解决浏览器对事件处理的兼容性方案。
  2. 简单方便把事件绑定到元素上。

缺点：

  1. 需要等待页面加载完毕后，才能绑定。
  2. 把事件绑定到所有选出来的元素上，当元素很多时，会影响性能问题。
  3. 不会绑定在它执行完后动态添加的元素上。

live()

委托对象document，为每个匹配元素的特定事件绑定事件处理函数。
语法：

  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • data：可选。传递给事件对象的额外数据对象。
  • function：定义当事件发生时运行的方法。

优点：

  1. 可以在document ready之前绑定。
  2. 动态添加的元素也可以触发早先绑定的事件，因为事件真正的绑定是在document上。
  3. 仅有一次的事件绑定，绑定到document上而不像.bind()那样给所有的元素挨个绑定。

缺点：

  1. 使用event.stopPropagation()是没用的，因为都要到达document。
  2. 使用matchSelector方法来选出那个事件被调用时，会非常慢，因为所有的selector/event都被绑定到document。
  3. 当发生事件的元素在DOM树中很深的时候，会有性能问题。
  4. Chaining没有被正确的支持。
  5. 从jQuery1.7开始已经不被推荐，在jQuery1.9中被移除了。

delegate()

自定义委托对象，为每个匹配元素的特定事件绑定事件处理函数。
语法：

  • chilidSelector：可选。一个选择器字符串用于过滤器的触发事件的选择器元素的后代。如果选择器为null或省略，当它到达选定的元素，事件总是触发。
  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • data：可选。传递给事件对象的额外数据对象。
  • function：定义当事件发生时运行的方法。

优点：

  1. 可以绑定在动态添加的元素上。
  2. 可以自由选择事件绑定的元素，有效的减小所要查找的元素。
  3. chaining被正确的支持了。

缺点：

  1. 需要查找哪个元素上发生了哪个事件，尽管比document少很多了，不过，还是得浪费时间来查找。

on()

自定义委托对象，为每个匹配元素的特定事件绑定事件处理函数。在jQuery 1.7中，.on()方法 提供绑定事件处理程序所需的所有功能。
语法：

  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • chilidSelector：可选。一个选择器字符串用于过滤器的触发事件的选择器元素的后代。如果选择器为null或省略，当它到达选定的元素，事件总是触发。
  • data：可选。传递给事件对象的额外数据对象。
  • function：该事件被触发时执行的函数。 false 值也可以做一个函数的简写，返回false。

语法：

注意：从jQuery1.7开始，jQuery引入了全新的事件绑定机制，on()和off()两个函数统一处理事件绑定。因为在此之前有bind(), live(), delegate()等方法来处理事件绑定，jQuery从性能优化以及方式统一方面考虑决定推出新的函数来统一事件绑定方法并且替换掉以前的方法。

trigger()

触发被选元素的指定事件类型。
语法：

  • EventType：定义绑定到匹配元素中的事件类型。如果有多个事件使用空格分隔。
  • data：可选。传递给事件对象的额外数据对象。

Event方法

preventDefault()

该方法将通知 Web 浏览器不要执行与事件关联的默认动作（如果存在这样的动作）。

例如，如果 type 属性是 “submit”，在事件传播的任意阶段可以调用任意的事件句柄，通过调用该方法，可以阻止提交表单。

注意，如果 Event 对象的 cancelable 属性是 fasle，那么就没有默认动作，或者不能阻止默认动作。无论哪种情况，调用该方法都没有作用。

stopPropagation()

终止事件在传播过程的捕获、目标处理或起泡阶段进一步传播。调用该方法后，该节点上处理该事件的处理程序将被调用，事件不再被分派到其他节点。

该方法将停止事件的传播，阻止它被分派到其他 Document 节点。在事件传播的任何阶段都可以调用它。注意，虽然该方法不能阻止同一个 Document 节点上的其他事件句柄被调用，但是它可以阻止把事件分派到其他节点。

return false

return false等效于同时调用e.preventDefault()和e.stopPropagation()
